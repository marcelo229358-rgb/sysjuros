# Sistema de Controle de Cobranças e Juros — Fase 6
## Docker + Deploy no VPS Hostinger

Pré-requisito: Fases 1–5 validadas. `npm run build` passando em backend e frontend.

---

## 1. Visão geral da infraestrutura

```
VPS Hostinger (KVM 2)
├── Nginx (reverse proxy, porta 80/443)
│   └── subdominio.seudominio.com.br → porta 3000 (backend)
│   └── subdominio.seudominio.com.br → porta 5173 (frontend, ou servido pelo Nginx)
├── Docker Compose
│   ├── backend (Node.js + Express) → porta 3000
│   ├── frontend (Nginx servindo build estático do React) → porta 80 interno
│   └── postgres → porta 5432 (interno, não exposto)
└── Evolution API (WhatsApp) → porta 8080
```

**Decisão:** usar Docker Compose pra backend + frontend + PostgreSQL, com Nginx do host como reverse proxy. A Evolution API roda em container separado (docker-compose próprio ou junto, você decide depois de instalar).

## 2. Dockerfile — Backend

```dockerfile
# backend/Dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine AS production
WORKDIR /app
ENV NODE_ENV=production

# puppeteer precisa de deps no Alpine
RUN apk add --no-cache \
  chromium \
  nss \
  freetype \
  harfbuzz \
  ca-certificates \
  ttf-freefont

# diz ao puppeteer pra usar o chromium do sistema, não baixar o próprio
ENV PUPPETEER_SKIP_CHROMIUM_DOWNLOAD=true
ENV PUPPETEER_EXECUTABLE_PATH=/usr/bin/chromium-browser

COPY package*.json ./
RUN npm ci --only=production
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/prisma ./prisma

EXPOSE 3000
CMD ["sh", "-c", "npx prisma migrate deploy && node dist/server.js"]
```

**Por que Alpine com Chromium manual:** o `puppeteer` por padrão baixa o Chrome inteiro (~200MB) no build. No Alpine, instalamos só o Chromium do repositório do sistema e apontamos o Puppeteer pra ele — imagem final muito menor.

## 3. Dockerfile — Frontend

```dockerfile
# frontend/Dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
ARG VITE_API_URL
ENV VITE_API_URL=$VITE_API_URL
RUN npm run build

FROM nginx:alpine AS production
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx-frontend.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

```nginx
# frontend/nginx-frontend.conf
server {
  listen 80;
  root /usr/share/nginx/html;
  index index.html;

  # React Router — redireciona tudo pro index.html
  location / {
    try_files $uri $uri/ /index.html;
  }

  gzip on;
  gzip_types text/plain text/css application/json application/javascript;
}
```

## 4. docker-compose.yml

```yaml
# docker-compose.yml (raiz do projeto)
version: '3.9'

services:
  postgres:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: ${POSTGRES_DB}
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - sysjuros_net
    # NÃO expõe porta pro host — só acessível internamente

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    restart: unless-stopped
    depends_on:
      - postgres
    environment:
      DATABASE_URL: postgresql://${POSTGRES_USER}:${POSTGRES_PASSWORD}@postgres:5432/${POSTGRES_DB}
      JWT_SECRET: ${JWT_SECRET}
      NODE_ENV: production
      EVOLUTION_API_URL: ${EVOLUTION_API_URL}
      EVOLUTION_API_KEY: ${EVOLUTION_API_KEY}
      EVOLUTION_INSTANCE: ${EVOLUTION_INSTANCE}
    ports:
      - "3000:3000"
    networks:
      - sysjuros_net

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
      args:
        VITE_API_URL: ${VITE_API_URL}
    restart: unless-stopped
    ports:
      - "4000:80"   # frontend servido pelo Nginx interno na porta 4000 do host
    networks:
      - sysjuros_net

volumes:
  postgres_data:

networks:
  sysjuros_net:
    driver: bridge
```

## 6. Nginx do host (reverse proxy)

```nginx
# /etc/nginx/sites-available/sysjuros
server {
    listen 80;
    server_name api.seudominio.com.br;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;

        # PDF pode demorar — aumentar timeout
        proxy_read_timeout 60s;
    }
}

server {
    listen 80;
    server_name app.seudominio.com.br;

    location / {
        proxy_pass http://localhost:4000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

Ativar e testar:
```bash
sudo ln -s /etc/nginx/sites-available/sysjuros /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

SSL com Certbot (após apontar DNS):
```bash
sudo certbot --nginx -d api.seudominio.com.br -d app.seudominio.com.br
```

## 7. Script de deploy (`deploy.sh`)

```bash
#!/bin/bash
set -e

echo "🚀 Iniciando deploy do SysJuros..."

# 1. atualizar código
git pull origin main

# 2. rebuild e restart dos containers
docker compose down
docker compose build --no-cache
docker compose up -d

# 3. aguardar backend subir (migration roda automaticamente no CMD do Dockerfile)
echo "⏳ Aguardando backend iniciar..."
sleep 10

# 4. verificar saúde
curl -f http://localhost:3000/health || echo "⚠️ Backend ainda não respondeu, verificar logs"

echo "✅ Deploy concluído!"
echo "Backend: https://api.seudominio.com.br"
echo "Frontend: https://app.seudominio.com.br"
```

Tornar executável:
```bash
chmod +x deploy.sh
```

## 9. Ordem de execução no VPS

```bash
# 1. clonar o projeto
git clone https://github.com/seu-usuario/sysjuros.git
cd sysjuros

# 2. criar .env.production com as variáveis reais
cp .env.example .env.production
nano .env.production

# 3. primeiro deploy
docker compose --env-file .env.production up -d --build

# 4. verificar logs
docker compose logs -f backend
docker compose logs -f frontend

# 5. configurar Nginx + SSL (seção 6 acima)

# deploys futuros
./deploy.sh
```
